Buttons

Description: Buttons allow the user to interact with the application. There are five states a button can be in, which will indicate to the user when or how to use the button. The five states are:

  • Active – Indicating that a user can select the button to perform an action
  • Selected – Indicating that a user already selected the button
  • Focus – Indicating that the user used the keyboard to navigate to the button
  • Hover – Indicating that the user used the mouse to hover over the button
  • Disabled – Indicating that the user cannot select the button to perform and action

508 Guidelines: Adding the title attribute to the button with helpful content will allow the screen reader to inform the user details about the button that requires user interaction.

TIP: The default and hover button state will appear the same for the eHMP application.

TIP: To disable a button, add the attribute "disabled".

Button with dropdown

Description: Split buttons have dual functionality. The text side allows users to type in content and the button triggers a drop down menu with options that users can select.

508 Guidelines: A text field uses the input field and the button uses a drop down menu, which will both follow the same guidelines as in the form elements section.
